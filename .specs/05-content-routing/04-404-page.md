# Spec 5.4: 404 Page

**Last Modified**: 2026-03-31
**Status**: Verified

## User Story

As a user, I want a helpful error page if I navigate to a non-existent URL, So that I can easily find my way back to the learning content.

## Context

### Why

A custom 404 page provides a better user experience than the default browser or Next.js error pages, keeping the user within the app's visual context and offering clear recovery paths.

### Related Specs

- Spec 4.5: Header Component

### Dependencies

- Spec 4.5: Header Component

## Technical Specification

### Components/Modules

**File**: `src/pages/404.tsx`
- Renders `<Header>`
- Renders a "404 - Page Not Found" message
- Provides links to Home (`/`) and Catalog (`/learn`)

## Acceptance Criteria

- [ ] **AC1**: Custom 404 rendering
  - Given a user navigates to an undefined route (e.g., `/does-not-exist`)
  - When Next.js handles the request
  - Then the custom `404.tsx` page is displayed instead of the default Next.js error

- [ ] **AC2**: Visual consistency
  - Given the 404 page is rendered
  - When inspected
  - Then it includes the standard global `<Header>` and uses the app's dark theme styling

- [ ] **AC3**: Recovery links
  - Given the 404 page is rendered
  - When the user looks for navigation options
  - Then there are clear, prominent links to "Go Home" and "Start Learning" (catalog)
